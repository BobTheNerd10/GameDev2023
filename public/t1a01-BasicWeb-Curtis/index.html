V stands for variables. Almost every programming language has variables, but many different programming languages handle variables differently.

Javascript has 3 ways to define variables. LET, VAR, and CONST

LET is the typical variable definition (let x = 2)

VAR is extremely similar to LET, but it automatically gets pushed to the top of the file (var x = 2).

CONST is *kind of* similar to LET. It's shorthand for "constant", and it stores values like LET, but it can NEVER BE CHANGED! (const x = 2)
I stands for input/output

Inputs are things that let the user interact with your website. Some inputs include text boxes, buttons, number inputs, and selections! Outputs are what you use to return data back to the user (for this class, that's typically using getElementById to change a div).
D stands for decisions. Decisions are if statements and else statements. "If" only runs the code if the specified condition is true. "Else" only runs the code if the specified condition from the "if" statement above it is false.
E stands for events. They're stuff like "onclick", "onhover", and "onblur". They trigger Javascript code to run whenever your webpage is interacted with in a specific way.
O stands for objects. Objects are variables that can store a bunch of different parameters. They're useful for defining specific instances of something if you want to give those instances a bunch of data.
F stands for functions. Functions store bits of Javascript code so they can be used repeatedly and make your code look nicer. They also accept parameters.
L stands for Loops. Loops are used to loop over a piece of Javascript. The three main kinds of loops in Javascript are "while" (which runs while the specified condition is true), "for" (which lets you use a counter to run code a certain amount of times), and "foreach" (which lets you iterate through arrays.)
A stands for Arrays. Arrays are useful for when you have a bunch of data that you need to group together, but you don't need to refer to the data by name (because otherwise objects are better.)
C stands for classes. Classes in Javascript are more or less just objects with stricter rules while also having the ability to store functions. You can also extend a class to base one class off of another class.
